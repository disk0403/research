# AGENTS.md

このファイルは、このリポジトリで作業する AI エージェント向けの運用指針である。リポジトリ全体に適用する。

## 最優先ルール

- ファイルの作成、編集、削除、移動、整形、生成物の書き出しなど、リポジトリ内のファイルを改変する操作は、ユーザーが明示的に変更を依頼した場合のみ行う。
- 明示的な変更依頼がない場合は、絶対に勝手にファイルを改変しない。
- 調査、説明、レビュー、方針相談、質問への回答を求められた場合は、読み取りと説明に徹する。
- 変更が必要だと判断しても、ユーザーが変更を依頼していない場合は、提案にとどめる。
- 変更する場合は、対象ファイルと意図を事前に把握し、無関係なファイルや既存のユーザー変更を巻き込まない。
- ファイルの内容を改変した場合は、関連する `.md` などのドキュメントも確認し、リポジトリ全体として整合性が取れるように必要な更新を行う。

## 作業前の確認

タスクを実行する前に、必要な範囲でリポジトリを俯瞰する。

- まず現在の作業ディレクトリ、対象ファイル、関連ディレクトリを確認する。
- ルートの `README.md`、対象プロジェクトの `README.md`、`ROADMAP.md`、関連する `papers/notes/` など、タスクに必要なドキュメントを読む。
- 既存の実装、命名、配置、依存関係、検証方法を確認してから作業する。
- `git status` で既存の変更を確認し、ユーザーや他の作業による変更を不用意に上書きしない。
- 外部由来の資産、ローカル環境、ログ、PDF、個人メモを研究コードとして扱わない。

## リポジトリの位置づけ

このリポジトリは、Unitree Go2 を中心とした四脚ロボット研究のための研究開発ワークスペースである。主な対象は MuJoCo シミュレーション、歩行制御、視覚・言語モデルを含むナビゲーション、HRI、安全評価、実機移行の準備である。

## ドキュメントの役割

- `README.md`: リポジトリ全体の入口。概要、主要ディレクトリ、代表的なプロジェクトへの案内を書く。
- `ROADMAP.md`: 研究方針、マイルストーン、安全方針、評価範囲、直近の実装計画を書く。
- `projects/*/README.md`: 各プロジェクトのセットアップ、実行方法、主要スクリプト、検証コマンド、前提資産を書く。
- `papers/references.bib`: 論文の BibTeX メタデータを書く。
- `papers/notes/`: 論文ごとの要約、研究との関係、使えるアイデア、限界を書く。

## ディレクトリ方針

```text
.
├── AGENTS.md
├── README.md
├── ROADMAP.md
├── papers/
│   ├── references.bib
│   ├── notes/
│   └── pdfs/
├── projects/
│   └── go2-mujoco/
│       ├── README.md
│       ├── requirements.txt
│       ├── scripts/
│       ├── external/
│       ├── logs/
│       └── .venv/
└── local/
```

- MuJoCo 固有のコード、設定、説明は `projects/go2-mujoco/` に置く。
- 研究コードは原則として対象プロジェクトの `scripts/` など、README に示された場所に置く。
- 論文メモや引用情報は `papers/` に置く。
- リポジトリ全体の研究方針は `ROADMAP.md` に集約する。
- 一時的なメモ、議論整理、スライド下書き、雑多な Markdown 出力、個人用作業ログは `local/` に置く。
- ディレクトリ構造が変わる場合は、関連する README と必要に応じてこのファイルを更新する。

## Git に入れるもの・入れないもの

Git に入れるもの:

- 研究コード。
- 再現に必要な小さな設定ファイル。
- `README.md`、`ROADMAP.md`、`AGENTS.md` などの運用・説明文書。
- 論文の BibTeX と Markdown メモ。

Git に入れないもの:

- `projects/go2-mujoco/.venv/`
- `projects/go2-mujoco/external/`
- `projects/go2-mujoco/logs/`
- `papers/pdfs/`
- `local/`
- `__pycache__/`
- `.DS_Store`
- `._*`
- `.env`
- 実行ログ、デバッグ画像、ローカルキャッシュ

外部資産やローカル環境は、必要に応じて README に取得方法や配置場所だけを書く。

## 外部資産とローカル環境

- `projects/go2-mujoco/external/` は外部由来のモデル、MuJoCo 資産、学習済みポリシーなどを置く場所である。
- 明示的な依頼がない限り、`external/` 配下は読み取り専用として扱う。
- `.venv/`、`logs/`、`papers/pdfs/`、`local/` は Git に入れないローカル領域として扱う。
- 外部資産を更新する必要がある場合は、配布元、ライセンス、取得方法、配置先を README に記録する。

## 論文を扱う場合

論文を追加・議論する場合は、原則として次を確認する。

- `papers/references.bib` に BibTeX メタデータを置く。
- `papers/notes/` に論文ごとの Markdown メモを置く。
- 論文タイトル、手法名、モデル名、データセット名など、識別子として意味がある英語はそのまま書く。
- PDF は `papers/pdfs/` に置けるが、明示的に依頼されない限り Git に入れない。

## 検証方針

- コードを変更した場合は、影響範囲に応じて関連 README に記載されたスモークテストや実行コマンドを確認する。
- 検証コマンドを実行できなかった場合は、理由を明記する。
- 実行ログやデバッグ画像は `logs/` や `local/` など、Git に入れない場所に置く。
- 検証手順そのものが変わった場合は、AGENTS ではなく対象 README を更新する。

## 安全・研究倫理

- 実機、人間、視覚障害者ユーザーを巻き込む前に、シミュレーションで危険行動を確認・低減する。
- 現段階の成果を「盲導犬の代替」として表現しない。
- 初期対象は屋内、低速、限定環境に絞る。
- 公道、横断歩道、駅、階段、混雑環境、視覚障害者協力者を含む評価は、倫理・安全レビュー、同意、安全担当者、緊急停止手順を整えてから扱う。
- ユーザーの停止、拒否、緊急停止を常に優先する設計を前提にする。

## 言語方針

- このリポジトリでの回答には日本語を使う。
- Markdown ファイルを書く・編集するときは日本語を使う。
- コマンド、コード、ファイル名、パッケージ名、論文の原題、モデル名、API 名など、識別子として意味がある英語は変更しない。

## AGENTS.md の更新方針

`AGENTS.md` は、AI エージェントに守らせるべき作業ルールが変わった場合にだけ更新する。

更新してよい例:

- ファイル改変、調査、検証、外部資産、安全方針などの運用ルールが変わった。
- リポジトリ全体に適用される作業上の注意が増えた。
- ディレクトリの役割やドキュメントの責務が変わった。

更新しない例:

- 個別スクリプトの細かい仕様説明。
- 一時的な実験結果。
- README に書けば十分なセットアップや実行方法。
- 単なる文章の好みや軽微な言い換え。
